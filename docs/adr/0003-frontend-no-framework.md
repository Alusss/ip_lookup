# ADR 0003: 前端不使用框架

## 状态

已采纳

## 上下文

前端页面结构极简：IP 展示 + IPv6 检测按钮 + 复制/刷新功能。不需要路由、状态管理、组件库。

## 决策

使用原生 HTML + CSS + JS (ES2020+)，不引入 React/Vue/Svelte 等框架。如需要构建，使用 Vite 仅做压缩与 hash。

## 理由

1. 页面交互极简，框架会增加 ~30KB+ 的 JS 开销
2. 无框架 = 无框架漏洞 = 更小的攻击面
3. 原生 JS 可直接操作 DOM，无虚拟 DOM 开销
4. Cloudflare Pages 可直接托管静态文件，无需构建步骤（或极简构建）

## 后果

- 需要手动管理 i18n（集中 `i18n.js`）
- 需要手动实现请求状态机（有限状态机模式）
- 无 HMR 等开发便利，但本项目页面结构简单不需要
